Aplicar --tar-bg como fondo base del <body> y del contenedor principal en PlayerShell.tsx.

Gradiente en la cabecera de playlist
Cuando el usuario abre una playlist (vista de detalle), el bloque superior con la portada y el nombre debe tener un fondo degradado que tome el color dominante de la portada y se desvanezca hacia abajo exactamente como lo hace Spotify en la imagen que enviaste. El agente debe implementarlo así en PlaylistsView.tsx:
css.playlist-hero {
  background: linear-gradient(
    to bottom,
    var(--tar-purple) 0%,      /* color dominante arriba */
    rgba(13, 31, 30, 0.6) 60%, /* transición */
    var(--tar-bg) 100%         /* fondo base abajo */
  );
}
Para hacerlo dinámico (que el color cambie según la portada de cada playlist), el agente debe usar la librería colorthief — npm install colorthief — que extrae el color dominante de una imagen. Al cargar la playlist, extraer el color dominante de la portada y usarlo como el primer stop del gradiente inline:
typescriptimport ColorThief from 'colorthief';

const thief = new ColorThief();
const img = new Image();
img.crossOrigin = 'anonymous';
img.src = playlist.coverUrl;
img.onload = () => {
  const [r, g, b] = thief.getColor(img);
  setHeroColor(`rgb(${r},${g},${b})`);
};
El style del hero queda como:
typescriptstyle={{
  background: `linear-gradient(to bottom, ${heroColor} 0%, rgba(13,31,30,0.6) 55%, #0D1F1E 100%)`
}}

Gradiente en NowPlayingView
En la vista "Reproduciendo ahora", el fondo debe hacer lo mismo: tomar el color dominante del artwork de la canción actual y degradar hacia abajo. Mismo patrón con colorthief, aplicado cada vez que cambia currentTrack.
